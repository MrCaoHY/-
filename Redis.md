### 什么是Redis

c语言开发的一个开源的高性能键值对数据库，官方提供测试数据，50个并发执行100000个请求，读的速度是110000次/s，写的速度是81000次/s，且Redis通过提供福哦中简直数据类型来适应不停场景下的存储需求，目前位置Redis支持的键值数据类型如下：

* 字符串类型 string
* 散列类型 hash
* 列表类型 list
* 集合类型 set
* 有序集合类型 sorted set

## Redis的应用场景

* 缓存（数据查询，短连接，新闻内容，商品内容等）
* 聊天室的在线好友
* 任务队列（秒杀，抢购，12306等）
* 应用排行榜
* 网站访问统计
* 数据过期处理（可以精确到毫秒）
* 分布式集群架构中的session分离

## 字符串类型String(Key-Value)

字符串类型是Redis最基础的数据结构，字符串类型可以是JSON、XML甚至是二进制的图片等数据，但是最大值不能超过512MB。

String是最常用的一种数据类型，普通的key/value存储都可以归为此类。
一个Key对应一个Value，string类型是二进制安全的。
Redis的string可以包含任何数据，比如jpg图片(生成二进制)或者序列化的对象。

### 字符串string常用命令

* set key value 添加/修改数据

```shell
127.0.0.1:6379> set company "itcast"
OK
127.0.0.1:6379> set name "itcast"

```

* get key 获取数据

```shell
127.0.0.1:6379> get company
"itcast"
```

* del key 删除数据

```shell
127.0.0.1:6379> get name
"itcast"
127.0.0.1:6379> del name
(integer) 1
127.0.0.1:6379> get name
(nil)

incr命令 自增

```

```shell
1) "name"
2) "redis tutorial"
3) "description"
4) "redis basic commands for caching"
5) "likes"
6) "20"
7) "visitors"
8) "23000"
```

一个列表最多可以包含 232 - 1 个元素 (4294967295, 每个列表超过40亿个元素)。

```
redis 127.0.0.1:6379> LPUSH runoobkey redis
(integer) 1

(integer) 2
redis 127.0.0.1:6379> LPUSH runoobkey mysql
(integer) 3
redis 127.0.0.1:6379> LRANGE runoobkey 0 10

2) "mongodb"
3) "redis"
```

### 集合set

Redis 的 Set 是 String 类型的无序集合。集合成员是唯一的，这就意味着集合中不能出现重复的数据。

集合对象的编码可以是 intset 或者 hashtable。



Redis 中集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是 O(1)。

集合中最大的成员数为 232 - 1 (4294967295, 每个集合可存储40多亿个成员)。

```shell
redis 127.0.0.1:6379> SADD runoobkey redis
(integer) 1
redis 127.0.0.1:6379> SADD runoobkey mongodb
(integer) 1
redis 127.0.0.1:6379> SADD runoobkey mysql
(integer) 1
redis 127.0.0.1:6379> SADD runoobkey mysql
(integer) 0
redis 127.0.0.1:6379> SMEMBERS runoobkey

1) "mysql"
2) "mongodb"
3) "redis"
```

### sorted set 有序集合

Redis 有序集合和集合一样也是 string 类型元素的集合,且不允许重复的成员。

不同的是每个元素都会关联一个 double 类型的分数。redis 正是通过分数来为集合中的成员进行从小到大的排序。

有序集合的成员是唯一的,但分数(score)却可以重复。

集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是 O(1)。 集合中最大的成员数为 232 - 1 (4294967295, 每个集合可存储40多亿个成员)。

```shell

(integer) 1

(integer) 1

(integer) 1
redis 127.0.0.1:6379> ZADD runoobkey 3 mysql
(integer) 0
redis 127.0.0.1:6379> ZADD runoobkey 4 mysql
(integer) 0
redis 127.0.0.1:6379> ZRANGE runoobkey 0 10 WITHSCORES

1) "redis"
2) "1"
3) "mongodb"
4) "2"
5) "mysql"
6) "4"
```

## Redis持久化

Redis为持久化提供了两种方式：

RDB（默认支持无需配置）：在指定的时间间隔能对你的数据进行快照存储。
AOF：以日志的形式记录每次对服务器写的操作,当服务器重启的时候会重新执行这些命令来恢复原始的数据。

### RDB持久化机制

```shell
#手动执行一次保存操作 save
127.0.0.1:6379> save
OK

#设置本地数据库文件名 默认dump.db
dbfilename filename
```
